**Task 1: `refresh_screener_universe`**
```python
@celery_app.task(bind=True, max_retries=3, default_retry_delay=60)
def refresh_screener_universe(self) -> None:
```
Logic:
1. Get BrapiClient instance
2. Discover all tickers: paginate through `/quote/list?limit=100&page=N` until no more pages. Collect ticker symbols.
3. For each ticker, call `brapi_client.fetch_fundamentals(ticker)`. Sleep 200ms (`time.sleep(0.2)`) between calls.
4. On 429 from brapi: exponential backoff (sleep 2^attempt * 60 seconds) up to 3 retries per ticker, then skip and log warning.
5. For each successful response, upsert into `screener_snapshots` using `pg_insert(...).on_conflict_do_update(index_elements=["ticker", "snapshot_date"], set_={...})`.
6. Map brapi response fields to model columns: `regularMarketPrice` -> `regular_market_price`, `regularMarketChangePercent` -> `regular_market_change_percent`, `regularMarketVolume` -> `regular_market_volume`, `marketCap` -> `market_cap`, `priceEarnings` -> `pl`, `priceToBook` -> `pvp`, `dividendYield` -> `dy`, `earningsPerShare` -> `ev_ebitda` (check actual brapi field name for EV/EBITDA), `shortName` -> `short_name`, `sector` -> `sector`.
7. Also write to Redis: `_SCREENER_PREFIX + ticker` with JSON of fundamentals, TTL = `_SCREENER_TTL`.
8. Use `get_sync_db_session(tenant_id=None)` for all DB writes. Commit per batch of 50 tickers (not per ticker, not entire run).
9. Log summary: `"Screener universe refreshed: {success_count}/{total_count} tickers updated, {fail_count} failed"`

**Task 3: `refresh_tesouro_rates`**
```python
@celery_app.task(bind=True, max_retries=3, default_retry_delay=60)
def refresh_tesouro_rates(self) -> None:
```
Logic:
1. Try ANBIMA first:
   a. `_get_anbima_credentials()` — fetch from AWS SM
   b. `_get_anbima_token(client_id, client_secret)` — OAuth2 token
   c. `_fetch_tesouro_from_anbima(token)` — GET from ANBIMA_TITULOS_URL with Bearer token
   d. Parse response: list of bond records with `tipo_titulo` (or similar), `data_vencimento`, `taxa_indicativa`, `pu` (unit price)
2. If ANBIMA fails (any exception), log WARNING and fall back to CKAN CSV:
   a. `requests.get(CKAN_CSV_URL, timeout=60)` — 13MB CSV
   b. Parse with `csv.DictReader(io.StringIO(resp.text), delimiter=";")`
   c. Filter rows to today's date only (column `Data Base` matches today in DD/MM/YYYY format) to avoid processing full history since 2002
   d. Map columns: `Tipo Titulo` -> bond type, `Data Vencimento` -> maturity, `Taxa Compra Manha` -> buy rate, `Taxa Venda Manha` -> sell rate, `PU Compra Manha` -> unit price
3. For each bond record, write to Redis:
   - Key: `_TESOURO_PREFIX + bond_code` where `bond_code` = `"{tipo_titulo}_{vencimento_YYYYMMDD}"` (e.g., `"Tesouro IPCA+ 2035_20350515"`)
   - Value: JSON with `tipo_titulo`, `vencimento`, `taxa_indicativa`, `pu`, `data_base`, `source` ("anbima" or "ckan")
   - TTL: `_TESOURO_TTL`
4. Log: `"Tesouro rates refreshed: {count} bonds from {source}"`
5. On exception in both ANBIMA and CKAN, call `self.retry(exc=exc)`.
