# Phase 4 Plan 02: Async Analysis Pipeline

## Objective

Build the full async job pipeline: POST /ai/analyze/{ticker} and POST /ai/analyze/macro return 202 + job_id immediately. Celery worker runs AI analysis in background. Results stored in DB (new `ai_analysis_jobs` table via Alembic migration). Frontend polls GET /ai/jobs/{job_id} until status = "completed". Rate limiting via slowapi + Redis.

## Context

- AI skill tasks from 04-01: `ai.run_asset_analysis`, `ai.run_macro_analysis`
- Celery result backend is Redis — task results expire after 1h (configured in celery_app.py)
- We need a DB table for persistent job storage (beyond 1h Redis TTL)
- User plan field: `"free"` cannot access AI — return 403 with upgrade message
- Rate limit: 5 AI requests per user per hour (enforced via Redis + slowapi)
- Auth via `get_current_user` + `get_authed_db` (existing pattern)

## Tasks

### Task 1: DB model + Alembic migration for ai_analysis_jobs
`type="auto"`

Create `app/modules/ai/models.py`:
- `AIAnalysisJob` model (imports Base from auth.models):
  ```
  id: str (UUID, PK)
  tenant_id: str (36, indexed)
  job_type: str (20) — "asset" | "macro" | "portfolio"
  ticker: str | None (20, nullable) — for asset analysis
  status: str (20) — "pending" | "running" | "completed" | "failed"
  result_json: str | None (Text, nullable) — JSON-serialized result
  error_message: str | None (500, nullable)
  created_at: DateTime
  completed_at: DateTime | None (nullable)
  ```

Create Alembic migration `0004_add_ai_analysis_jobs.py`:
- `op.create_table("ai_analysis_jobs", ...)` with all columns
- Add index on `(tenant_id, status)`
- Add index on `(tenant_id, created_at DESC)` for job listing

Done criteria: model file and migration file exist; `alembic check` passes (or equivalent import check).

### Task 2: Pydantic schemas + AI router skeleton
`type="auto"`

Create `app/modules/ai/schemas.py`:
- `AnalysisRequest(BaseModel)` — `ticker: str` (for asset analysis)
- `MacroAnalysisRequest(BaseModel)` — no fields (uses authenticated user's portfolio)
- `JobResponse(BaseModel)` — `id, job_type, ticker, status, created_at, completed_at`
- `JobResultResponse(JobResponse)` — adds `result: dict | None`

Create `app/modules/ai/router.py`:
- `router = APIRouter()`
- `POST /ai/analyze/{ticker}` — accepts AnalysisRequest, dispatches `run_asset_analysis` Celery task, saves pending job to DB, returns 202 `JobResponse`
- `POST /ai/analyze/macro` — dispatches `run_macro_analysis`, saves pending job, returns 202 `JobResponse`
- `GET /ai/jobs/{job_id}` — returns `JobResultResponse`; if completed, parses result_json; if pending/running, returns status only
- `GET /ai/jobs` — list last 10 jobs for current tenant
- Premium gate on POST endpoints: `if current_user["plan"] == "free": raise HTTPException(403, "Upgrade to Premium to access AI analysis")`
- Rate limit: `@limiter.limit("5/hour")` on POST endpoints

Register router in `main.py`: `app.include_router(ai_router, prefix="/ai", tags=["ai"])`

Done criteria: router.py exists with 4 endpoints, 403 for free plan on POST, registered in main.py.

### Task 3: Celery task result callback — write to DB
