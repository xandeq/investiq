# Phase 4 Plan 01: AI Skills Adapters

## Objective

Build encapsulated AI skill adapters as Celery tasks: `dcf.py`, `valuation.py`, `earnings.py`, `macro.py`. Each adapter fetches market data from Redis/DB, calls OpenAI (via AWS Secrets Manager), and returns structured results. Register tasks in celery_app.py. All calls go through a resilient provider function with OpenRouter fallback.

## Context

- Existing Celery infrastructure: `app/celery_app.py`, workers use psycopg2 sync engine
- Market data in Redis: `market:fundamentals:{TICKER}`, `market:macro:*`, `market:historical:{TICKER}`
- AWS SM for OpenAI key: `tools/openai` → `OPENAI_API_KEY`; fallback `tools/openrouter` → `OPENROUTER_API_KEY`
- Existing `app/modules/portfolio/service.py` has `calculate()` skeleton for EXT-03
- User model has `plan` field ("free" | "pro") for premium gating

## Tasks

### Task 1: AI provider client with AWS SM + fallback
`type="auto"`

Create `app/modules/ai/provider.py`:
- `async def call_llm(prompt: str, system: str = "", model: str = "gpt-4o-mini") -> str`
- Fetch `OPENAI_API_KEY` from AWS SM (`tools/openai`) on first call, cache in module-level var
- Use `httpx.AsyncClient` POST to `https://api.openai.com/v1/chat/completions`
- On 402/429/insufficient_quota: auto-fallback to OpenRouter (`tools/openrouter`, model `deepseek/deepseek-chat`)
- Log which provider was used
- Raise `AIProviderError` (custom exception in same file) if all providers fail
- Create `app/modules/ai/__init__.py`

Done criteria: `provider.py` exists, `call_llm` defined, fallback logic present, custom exception defined.

### Task 2: Four skill adapters as pure async functions
`type="auto"`

Create `app/modules/ai/skills/`:
- `__init__.py`
- `dcf.py` — `async def run_dcf(ticker: str, fundamentals: dict, macro: dict) -> dict`
  - Builds a concise prompt: P/L, P/VP, DY, EV/EBITDA + SELIC/CDI/IPCA as discount rate context
  - Returns `{"ticker": str, "analysis": str, "methodology": "DCF", "disclaimer": DISCLAIMER_TEXT}`
- `valuation.py` — `async def run_valuation(ticker: str, fundamentals: dict) -> dict`
  - Relative valuation prompt: how does P/L and P/VP compare to sector norms?
  - Returns `{"ticker": str, "analysis": str, "methodology": "Valuation Relativa", "disclaimer": DISCLAIMER_TEXT}`
- `earnings.py` — `async def run_earnings(ticker: str, fundamentals: dict) -> dict`
  - Earnings quality + payout analysis from DY and fundamentals
  - Returns `{"ticker": str, "analysis": str, "methodology": "Análise de Lucros", "disclaimer": DISCLAIMER_TEXT}`
- `macro.py` — `async def run_macro_impact(macro: dict, allocation: list[dict]) -> dict`
  - How does current SELIC/IPCA/PTAX affect this specific portfolio allocation mix?
  - Returns `{"analysis": str, "methodology": "Impacto Macro", "disclaimer": DISCLAIMER_TEXT}`

DISCLAIMER_TEXT constant (defined in `app/modules/ai/skills/__init__.py`):
`"Análise informativa — não constitui recomendação de investimento (CVM Res. 19/2021)"`

All four functions call `call_llm` from `provider.py`.

Done criteria: 4 skill files exist, each returns dict with `disclaimer` key.

### Task 3: Celery tasks wrapping each skill
`type="auto"`

Create `app/modules/ai/tasks.py`:
- Import all 4 skill functions
- `@celery_app.task(name="ai.run_asset_analysis")` — `def run_asset_analysis(job_id: str, ticker: str, tenant_id: str) -> dict`
  - Sync wrapper: use `asyncio.run()` to call async skills
  - Fetches fundamentals from Redis sync (via `redis.Redis` sync client from REDIS_URL env)
  - Runs dcf, valuation, earnings in sequence
  - Returns `{"job_id": job_id, "ticker": ticker, "dcf": ..., "valuation": ..., "earnings": ..., "completed_at": ISO str}`
- `@celery_app.task(name="ai.run_macro_analysis")` — `def run_macro_analysis(job_id: str, tenant_id: str, allocation: list) -> dict`
  - Sync wrapper for macro skill
  - Returns `{"job_id": job_id, "macro": ..., "completed_at": ISO str}`

Register tasks in `celery_app.py` include list: add `"app.modules.ai.tasks"`.

Done criteria: `tasks.py` exists, 2 tasks defined, celery_app.py include list updated.
