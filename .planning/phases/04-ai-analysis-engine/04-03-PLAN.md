# Phase 4 Plan 03: AI Results UI

## Objective

Build the AI analysis frontend: an `/ai` page where pro users can request asset analysis (by ticker) and macro portfolio analysis. Results arrive via polling. Every AI output displays the mandatory CVM disclaimer. Free users see a locked preview with upgrade CTA. All components use 'use client'.

## Context

- Backend endpoints from 04-02: POST /ai/analyze/{ticker}, POST /ai/analyze/macro, GET /ai/jobs/{id}, GET /ai/jobs
- TanStack Query v5 for polling (`refetchInterval` until status = "completed")
- Existing auth pattern: `useAuth` hook, `plan` field available from /me endpoint
- Existing UI components: shadcn Card, Skeleton (already in project)
- Next.js App Router page at `app/ai/page.tsx`
- All components with state/hooks must have `'use client'` at top

## Tasks

### Task 1: AI API types and client functions
`type="auto"`

Create `frontend/src/features/ai/types.ts`:
- `AnalysisJob { id: string; job_type: string; ticker: string | null; status: "pending" | "running" | "completed" | "failed"; created_at: string; completed_at: string | null; result: AnalysisResult | null }`
- `AnalysisResult { dcf?: SkillResult; valuation?: SkillResult; earnings?: SkillResult; macro?: MacroResult }`
- `SkillResult { ticker: string; analysis: string; methodology: string; disclaimer: string }`
- `MacroResult { analysis: string; methodology: string; disclaimer: string }`

Create `frontend/src/features/ai/api.ts`:
- `requestAssetAnalysis(ticker: string): Promise<AnalysisJob>` — POST /ai/analyze/{ticker}
- `requestMacroAnalysis(): Promise<AnalysisJob>` — POST /ai/analyze/macro
- `getJob(jobId: string): Promise<AnalysisJob>` — GET /ai/jobs/{jobId}
- `listJobs(): Promise<AnalysisJob[]>` — GET /ai/jobs

Done criteria: types.ts and api.ts exist, all 4 functions exported.

### Task 2: TanStack Query hooks for AI
`type="auto"`

Create `frontend/src/features/ai/hooks/`:
- `useAnalysisJob.ts` — `useQuery({ queryKey: ['ai', 'job', jobId], queryFn: () => getJob(jobId), refetchInterval: (query) => query.state.data?.status === 'completed' || query.state.data?.status === 'failed' ? false : 2000, enabled: !!jobId })`
- `useJobList.ts` — `useQuery({ queryKey: ['ai', 'jobs'], queryFn: listJobs, staleTime: 30_000 })`

Done criteria: 2 hook files exist with correct polling logic.

### Task 3: AI UI components
`type="auto"`

Create `frontend/src/features/ai/components/`:

**DisclaimerBadge.tsx** (no 'use client' needed — pure display):
- Renders: `"Análise informativa — não constitui recomendação de investimento (CVM Res. 19/2021)"`
- Styled as amber/yellow badge — visually prominent, never hidden
- Must appear on every AI result panel

**SkillResultCard.tsx** ('use client'):
- Props: `{ result: SkillResult; title: string }`
- Renders: Card with title, methodology badge, analysis text (preserve whitespace/newlines), DisclaimerBadge at bottom
- Skeleton state when result is null/loading

**MacroResultCard.tsx** ('use client'):
- Props: `{ result: MacroResult }`
- Similar to SkillResultCard but labeled "Impacto Macro no Portfólio"
- DisclaimerBadge at bottom

**AnalysisRequestForm.tsx** ('use client'):
- Ticker input + "Analisar" button
- On submit: calls `requestAssetAnalysis(ticker)`, stores returned job_id in local state
- Shows spinner while job status is pending/running
- When job completes: renders SkillResultCard for DCF, Valuation, Earnings
- Error state for failed jobs

**PremiumGate.tsx** (no 'use client'):
- Props: `{ children: React.ReactNode }`
- If user plan === "free": show blurred preview overlay + upgrade CTA button (links to /planos)
- If user plan === "pro": render children normally
- Fetch user plan from /me endpoint (use existing useAuth hook or similar)

Done criteria: 5 component files exist, DisclaimerBadge renders CVM text, PremiumGate blocks free users.
