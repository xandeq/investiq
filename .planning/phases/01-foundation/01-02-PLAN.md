  <behavior>
    AUTH-01 — Registration:
    - POST /auth/register {email, password} → 201 {"message": "Verification email sent", "user_id": uuid}
    - Duplicate email → 409 {"error": "Email already registered"}
    - Weak password (< 8 chars) → 422
    - Password is stored as bcrypt hash — never plaintext

    AUTH-02 — Email verification:
    - POST /auth/login before verification → 403 {"error": "Email not verified", "is_verified": false}
    - GET /auth/verify-email?token=... with valid token → 200 {"message": "Email verified"}
    - GET /auth/verify-email?token=... with expired token (> 24h) → 400 {"error": "Verification link expired"}
    - POST /auth/resend-verification limited to 3 requests/hour per user (Redis INCR/EXPIRE, no slowapi dependency)

    AUTH-03 — Login and session:
    - POST /auth/login {email, password} → 200, sets access_token cookie (httpOnly, 15min) + refresh_token cookie (httpOnly, 7day, path=/auth/refresh)
    - POST /auth/refresh with valid refresh_token cookie → 200, rotates tokens (old marked used, new issued)
    - POST /auth/refresh with already-used refresh token → 401 + revokes ALL user refresh tokens (reuse detection)
    - POST /auth/logout → 200, clears both cookies

    AUTH-04 — Password reset:
    - POST /auth/forgot-password {email} → 200 {"message": "If that email exists, a reset link was sent"} (always 200, no user enumeration)
    - GET /auth/reset-password?token=... with valid token → 200
    - POST /auth/reset-password {token, new_password} → 200, password changed, all refresh tokens revoked

    EXT-03 — Email adapter swappable:
    - AuthService.__init__ accepts optional email_sender callable
    - Tests inject a stub: lambda to_email, subject, html: None
    - Production passes Brevo httpx implementation
  </behavior>
  <implementation>
    **Security primitives (backend/app/core/security.py):**
    - Generate RS256 key pair: `openssl genrsa -out /tmp/private_key.pem 2048 && openssl rsa -in /tmp/private_key.pem -pubout -out /tmp/public_key.pem` — store in AWS SM as tools/investiq-jwt, inject via env at runtime. NEVER commit key files.
    - `create_access_token(sub, tenant_id)` — PyJWT encode RS256, 15min exp, type="access"
    - `create_refresh_token(sub)` — PyJWT encode RS256, 7day exp, type="refresh"
    - `decode_token(token)` — PyJWT decode RS256, raises jwt.ExpiredSignatureError / jwt.InvalidTokenError
    - `hash_password(plain)` — passlib[bcrypt] CryptContext, bcrypt scheme
    - `verify_password(plain, hashed)` — passlib[bcrypt] verify
    - `set_auth_cookies(response, access_token, refresh_token)` — httpOnly=True, secure=True (in production), samesite="lax", refresh token scoped to path="/auth/refresh"
    - `get_current_user(request)` — FastAPI dependency: reads access_token cookie, decodes JWT, returns user_id + tenant_id

    **Models (backend/app/modules/auth/models.py):**
    - `User` table: id (UUID PK), tenant_id (UUID, not null — same as id for v1 one-tenant-per-user), email (unique not null), hashed_password, is_verified (bool default false), plan (String default "free" — EXT-02 DB-configurable), created_at, updated_at
    - `RefreshToken` table: id (UUID PK), user_id (FK User.id), token_hash (SHA256 of token, not plaintext), status (Enum: active/used/revoked), expires_at, created_at
    - `VerificationToken` table: id (UUID PK), user_id (FK), email (for resend lookup), token_hash, expires_at, created_at

    Note: tenant_id = user.id for v1 (one portfolio per user = one tenant per user). This is a v1 simplification — documented in code comment. Multi-user tenants are Phase 2+ concern.

    **Service (backend/app/modules/auth/service.py):**
    - All email sending goes through injected `email_sender` callable — never calls Brevo directly
    - Production email_sender uses Brevo httpx pattern from RESEARCH.md Pattern 4
    - Brevo API key fetched from AWS SM at startup: `python -m awscli secretsmanager get-secret-value --secret-id tools/brevo --query SecretString --output text` → parse JSON → BREVO_API_KEY, BREVO_FROM_EMAIL, BREVO_FROM_NAME
    - Store keys in settings (loaded via pydantic-settings from env) — Docker Compose injects them
    - Rate limiting for resend: Redis `INCR resend:{user_id}` with `EXPIRE 3600` if new key

    **Router (backend/app/modules/auth/router.py):**
    - All endpoints under `/auth` prefix
    - Register router in `app/main.py`: `app.include_router(auth_router, prefix="/auth", tags=["auth"])`
