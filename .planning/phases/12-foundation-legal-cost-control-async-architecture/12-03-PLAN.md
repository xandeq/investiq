Implement the async Celery task infrastructure, LLM provider fallback chain, and cache invalidation for analysis jobs.

Purpose: This is the execution engine — when a user requests analysis, a Celery worker picks it up, calls LLM providers with fallback, stores versioned results, and logs costs. Without this, the /analysis/dcf endpoint from Plan 02 creates jobs but nothing processes them.

Implement `async def call_analysis_llm(prompt: str, max_tokens: int = 300) -> tuple[str, dict]`:
- Iterate through ANALYSIS_LLM_CHAIN
- For each provider, wrap call in `asyncio.wait_for(..., timeout=config["timeout_seconds"])`
- OpenRouter: Use existing `call_llm` from `app.modules.ai.provider` with `system="You are a financial analyst. Be concise and factual."`, `tier="paid"`, `max_tokens=max_tokens`
- Groq: Direct httpx POST to `https://api.groq.com/openai/v1/chat/completions` with `os.environ.get("GROQ_API_KEY")` auth, model from config, temperature=0.7
- On success, return `(response_text, {"provider_used": config["provider"], "model": config["model"], "success": True})`
- On TimeoutError or any Exception, log warning and continue to next provider
- If all fail, raise `AIProviderError("All analysis LLM providers exhausted")`

Also implement `_get_cached_analysis_with_outdated_badge(ticker: str, analysis_type: str) -> dict | None`:
- Query most recent completed AnalysisJob for this ticker+type
- If found, parse result_json, add `"_outdated": True` and `"_outdated_reason": f"Analysis LLM quota exhausted. Results from {job.completed_at}. Contact support to refresh."`
- If not found, return None

Task steps (from RESEARCH.md Section 4.2):
1. Check quota via `_check_and_increment_quota(tenant_id)` — if False, `_update_job(job_id, "failed", error="Analysis quota exhausted")`, return
2. `_update_job(job_id, "running")`
3. Record `start_time = time.time()`
4. Fetch fundamentals — for Phase 12, create a `_fetch_fundamentals_stub(ticker: str) -> dict` that returns hardcoded sample data (real BRAPI integration comes in Phase 13). Return dict with: `current_price=27.80`, `eps=3.50`, `pe_ratio=7.94`, `revenue=500_000_000_000`, `free_cash_flow=50_000_000_000`, `dividend_yield=0.08`, `market_cap=400_000_000_000`
5. Calculate DCF — create `_calculate_dcf_stub(ticker, fundamentals, assumptions) -> dict` that returns: `fair_value=28.50`, `low=26.00`, `high=31.00`, `upside_pct=2.5`, `growth_rate=assumptions.get("growth_rate", 0.05)`, `discount_rate=assumptions.get("discount_rate", 0.10)`, `terminal_growth=assumptions.get("terminal_growth", 0.03)`. Real calculation comes in Phase 13.
6. Call LLM for narrative via `asyncio.run(call_analysis_llm(prompt, max_tokens=300))` — on AIProviderError, check for cached version via `_get_cached_analysis_with_outdated_badge`, if None use static fallback string
7. Build result dict with: ticker, fair_value, fair_value_range, current_price, upside_pct, assumptions, narrative, data_version_id (from `build_data_version_id()`), data_timestamp (ISO string), data_sources (from `get_data_sources()`), disclaimer (CVM_DISCLAIMER_SHORT_PT)
8. `_update_job(job_id, "completed", result_json=json.dumps(result))`
9. `log_analysis_cost(tenant_id, job_id, "dcf", ticker, duration_ms=int((time.time()-start_time)*1000), status="completed", llm_provider=llm_meta.get("provider_used"), llm_model=llm_meta.get("model"), input_tokens=..., output_tokens=...)`

Wrap steps 4-9 in try/except. On any exception:
- `_update_job(job_id, "failed", error=str(exc)[:500])`
- `log_analysis_cost(tenant_id, job_id, "dcf", ticker, duration_ms=..., status="failed")`

**invalidation.py** (`backend/app/modules/analysis/invalidation.py`):

Implement `async def on_earnings_release(ticker: str, filing_date: datetime) -> int`:
- Use `get_superuser_sync_db_session()` to update all AnalysisJob rows where:
  - `ticker == ticker`
  - `completed_at < filing_date`
  - `status == "completed"`
- Set `status = "stale"`, `error_message = "New earnings released; please refresh"`
- Clear Redis cache key `f"analysis:cache:{ticker}"` using `sync_redis.from_url(os.environ.get("REDIS_URL", "redis://redis:6379/0")).delete(key)`
- Return count of invalidated rows
- Log: `logger.info(f"Invalidated {count} analyses for {ticker} due to earnings release")`

Implement `def get_analyzed_tickers_recent_7d() -> list[str]`:
- Query distinct tickers from AnalysisJob where `created_at > utcnow - 7 days` and `status in ("completed", "stale")`
- Return list of ticker strings
