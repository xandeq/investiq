Create `backend/app/modules/analysis/sector.py` with a single function `calculate_sector_comparison(target_fundamentals: dict, peer_fundamentals: list[dict], ticker: str) -> dict`.

**How to find peers:** BRAPI does not have a "list stocks by sector" endpoint. Instead, use this approach:
1. The target ticker's `sector_key` comes from `fetch_fundamentals(ticker)["sector_key"]` (e.g., "energy", "financial-services", "basic-materials").
2. Maintain a hardcoded dict `_SECTOR_TICKERS` in sector.py mapping common sector_keys to well-known B3 tickers. Cover at least these sectors:
   - `"energy"`: ["PETR4", "PETR3", "CSAN3", "PRIO3", "RECV3", "RRRP3", "UGPA3", "VBBR3"]
   - `"financial-services"`: ["ITUB4", "BBDC4", "BBAS3", "SANB11", "BPAC11", "BRSR6", "ABCB4", "BMGB4"]
   - `"basic-materials"`: ["VALE3", "SUZB3", "KLBN11", "GGBR4", "CSNA3", "USIM5", "GOAU4", "BRAP4"]
   - `"utilities"`: ["ELET3", "SBSP3", "CPFE3", "ENGI11", "CMIG4", "TAEE11", "EGIE3", "EQTL3"]
   - `"consumer-defensive"`: ["ABEV3", "NTCO3", "PCAR3", "CRFB3", "ASAI3", "MDIA3", "SLCE3"]
   - `"consumer-cyclical"`: ["MGLU3", "LREN3", "ARZZ3", "VIVT3", "RENT3", "LCAM3", "ALPA4"]
   - `"healthcare"`: ["RDOR3", "HAPV3", "FLRY3", "QUAL3", "HYPE3", "RADL3"]
   - `"technology"`: ["TOTS3", "LWSA3", "CASH3", "POSI3", "INTB3"]

   - `"communication-services"`: ["VIVT3", "TIMS3", "OIBR3"]
3. If sector_key not found in dict, return early with an error result (sector not mapped).
4. Exclude the target ticker from peers. Limit to `max_peers`.

**Comparison logic:**
- For each peer ticker, call `fetch_fundamentals(peer)` (already Redis-cached 24h). Wrap in try/except DataFetchError -- skip peers that fail.
- Extract for each peer: ticker, current_price, pe_ratio, price_to_book, dividend_yield, roe, market_cap, profit_margins.
- Calculate sector averages and medians for: pe_ratio, price_to_book, dividend_yield, roe.
- For the target ticker, calculate percentile rank within peers for each metric.
- Build `data_completeness` showing how many peers had data vs how many were attempted.

**Return dict structure:**
```python
{
    "ticker": str,
    "sector": str,
    "sector_key": str,
    "peers_found": int,
    "peers_attempted": int,
    "max_peers": int,
    "target_metrics": {"pe_ratio": ..., "price_to_book": ..., "dividend_yield": ..., "roe": ...},
    "sector_averages": {"pe_ratio": ..., "price_to_book": ..., "dividend_yield": ..., "roe": ...},
    "sector_medians": {"pe_ratio": ..., "price_to_book": ..., "dividend_yield": ..., "roe": ...},
    "target_percentiles": {"pe_ratio": ..., "price_to_book": ..., "dividend_yield": ..., "roe": ...},
    "peers": [{"ticker": str, "pe_ratio": ..., "price_to_book": ..., "dividend_yield": ..., "roe": ..., "market_cap": ..., "current_price": ...}, ...],
    "data_completeness": {"peers_with_data": int, "peers_without_data": int, "missing_tickers": [...], "note": "X of Y peers included; Z missing data"},
}
```

**For median calculation:** Use `statistics.median()`. Filter out None values before calculating averages/medians.

**For percentile rank:** Count how many peers have a value less than or equal to the target, divide by total peers with data, multiply by 100. For P/E, lower percentile = cheaper = better.
